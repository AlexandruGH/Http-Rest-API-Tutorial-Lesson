Ghid de Design CRUD

Regulile de aur pentru un API curat (Actualizat 2024).

{/* 1. Naming & Nouns */}

1. Naming & Convenții

{/* Rule: Domain Model */}
Modelează Domeniul Real

Folosește termeni simpli și comuni. Nu inventa cuvinte complicate.

✅ /people ❌ /special-snowflakes
{/* Rule: Lowercase */}
Lowercase (Obligatoriu)

URL-urile sunt case-sensitive pe unele servere. Evită ambiguitatea.

✅ /items ❌ /Items
{/* Rule: Hyphens */}
Cratime, nu Underscores

Când URL-urile sunt link-uri subliniate, caracterul `_` devine invizibil.

✅ /legal-documents ❌ /legal_documents
{/* Rule: Implementation Hiding */}
Ascunde Implementarea

Nu arăta utilizatorului ce tehnologie folosești (.php, .jsp).

  • /api.php/users
  • /api/users
{/* Rule: Plurals */}
Substantive la Plural

Folosește pluralul pentru consistență, chiar și dacă returnezi un singur element.

  • /file
  • /files
{/* 2. Path Structure */}

2. Structura Căilor (Paths)

Identificatori în Path, nu Query

Resursele individuale se identifică prin URL, nu prin parametri.

✅ /contacts/22 ❌ /contacts?id=22
Limită de Nesting

Maxim 2 niveluri de adâncime. Dacă e mai adânc, e prea complex.

✅ /users/1/addresses ❌ /users/1/company/5/department/3
{/* 3. HTTP Methods */}

3. Metode HTTP & Nuanțe

PUT vs PATCH

PUT = Înlocuire Totală (Rescriere).
PATCH = Modificare Parțială.

GET vs HEAD

GET = Descarcă tot (Headers + Body).
HEAD = Doar verifică (Headers only). Util pentru cache/verificare existență.

{/* 4. Query Params (Restored) */}

4. Filtrare, Sortare & Paginare

Nu crea endpoint-uri noi pentru căutare (ex: /searchUsers). Folosește Query Parameters.

Căutare GET /products?search=iphone
Paginare GET /products?page=2&limit=10
Sortare GET /products?sort=price_desc
{/* 5. Advanced & Controversial */}

5. Subiecte Avansate / Controversate

{/* Versioning */}

Versioning

Abordarea Pragmatică (URL) /v1/users

Ușor de văzut, ușor de folosit. Standard în industrie.

Abordarea "Puristă" (Header) Accept-Version: v1

Păstrează URL-ul curat, dar e mai greu de testat în browser.

Resource Types (Extensii)

Să demistificăm API-urile

Ai auzit de REST și HTTP, dar te încurcă termenii? Această lecție îți arată vizual cum funcționează comunicarea web modernă.

HTTP

HyperText Transfer Protocol

Este mecanismul de transport. Este camionul care cară marfa. Definește CUM vorbesc calculatoarele (folosind Verbe, Headere, Coduri de Status). HTTP nu îi pasă ce transporți sau cum e structurat URL-ul.

REST

Representational State Transfer

Este un set de reguli (stil). Îți spune cum să folosești HTTP corect. Regula de aur: Gândește în Resurse (substantive), nu în Acțiuni (verbe).

Analogie Simplă

HTTP este Limba Română (vocabularul, gramatica).
REST este stilul Jurnalistic (reguli despre cum să scrii un articol clar).
Poți vorbi română (HTTP) fără să fii jurnalist (REST), dar rezultatul poate fi haotic.

Verbe HTTP (Metode)

  • 1xx - Informațional (100 Continue)
  • 2xx - Succes (200 OK, 201 Created)
  • 3xx - Redirectare (301 Moved)
  • 4xx - Eroare Client (404 Not Found, 400 Bad Req)
  • 5xx - Eroare Server (500 Internal Error)
